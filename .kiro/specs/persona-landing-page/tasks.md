# Implementation Plan: persona-landing-page

## Task Format

- 要件 ID は数値のみ（例: 1.1, 2.2）。説明や括弧は付けない。
- `(P)` は他タスクと並行実行可能な場合に付与。順序前提のタスクには付けない。

- [ ] 1. Tailwind CSS を npm 化し CDN を廃止する
- [x] 1.1 フロントエンドに Tailwind と Vite 用プラグインを追加し、ビルド時にスタイルがバンドルされるようにする
  - パッケージ追加と Vite 設定にプラグインを追加する
  - メインの CSS に Tailwind のインポートを追加する
  - _Requirements: 1.1_
- [x] 1.2 index.html から Tailwind の CDN スクリプトを削除する
  - 既存のインラインスタイル（フォント・背景・スクロールバー）は維持する
  - _Requirements: 1.1_
- [x] 1.3 ビルドと開発サーバーで既存画面（ログイン、Genesis、ダッシュボード）の見た目が変わっていないことを確認する
  - _Requirements: 1.1_

- [ ] 3. 未認証時にランディングとフォームを切替える状態を App に追加する

- [ ] 4. ランディングページの内容と体験を実装する

- [ ] 4.4 モバイル・タブレット・デスクトップで表示が崩れないようにし、フォーカス順序とセマンティックな構造でアクセシビリティを満たす
