## 世界観テーマ定義（Arcane / Chronicle / Neo）

Skill Quest AI のUI全体を貫く「世界観テーマ」を3つに整理したドキュメントです。  
`CharacterProfile.worldviewId` と `WORLDVIEWS` に対応するビジュアル・体験トーンの基準として利用します。

---

## 1. Arcane Terminal（アーケイン・ターミナル）

- **キーワード**: 近未来ハッカー / 魔導ターミナル / ネオン / コードの海
- **メインの感情トーン**: クール・ストイック・集中、静かな高揚感
- **主な色の方向性**:
  - 背景: 濃いネイビー〜ブラック寄りダークブルー
  - サーフェス（カード）: ネイビー〜ダークグレー
  - アクセント: シアン / パープル（ネオン感のあるブルー〜バイオレット）
- **世界観イメージ**:
  - コマンドラインやHUDが浮かぶ魔導端末の中で、自分だけのクエストログをハックしていく感覚。
  - ターミナルに呪文（コマンド）を打ち込むように、日々のタスクを処理していく。
- **UI・演出の方向性**:
  - 線的・エッジの効いたシェイプ（角丸は控えめ、直線的な区切り）。
  - グローやアウトラインでネオン感を出しつつも、可読性を損なわない程度に。
  - 重要なアクション（CTA）はシアン〜パープル系の強いアクセントで「押したくなる」ボタンに。
- **体験としての位置付け**:
  - 「静かなコントロールルーム」で、自分の成長を淡々と積み上げたいユーザーにフィット。
  - 既存のデフォルトテーマに最も近く、世界観未設定ユーザーの初期値として扱う。

---

## 2. Chronicle Campus（クロニクル・キャンパス）

- **キーワード**: 夜のキャンパス / 図書館 / 研究棟 / ワインレッド
- **メインの感情トーン**: 温かい知性・落ち着き・少しノスタルジック
- **主な色の方向性**:
  - 背景: 暗めのボルドー〜深いワインレッド
  - サーフェス（カード）: ダークブラウン / レッドブラウン
  - アクセント: ピンク〜ローズゴールド / 柔らかいゴールド
- **世界観イメージ**:
  - 夜の大学キャンパスで、自習室や図書館にこもって黙々と学ぶ時間。
  - グリモワールや書架に囲まれた「研究ノート」としてタスク・ナラティブを積み上げていく。
- **UI・演出の方向性**:
  - 若干丸みのあるカードと、紙・装丁を連想させる微妙なテクスチャ感（※実装はフラットだが印象として）。
  - 見出しやラベルにゴールド寄りのアクセントを使い、「章タイトル」「小見出し」のような雰囲気を出す。
  - プライマリボタンはワインレッド〜ピンク寄りの落ち着いた色で、勉強アプリとして違和感のないトーンに。
- **体験としての位置付け**:
  - 「学び」「資格勉強」など、勉強系クエストと相性が良い世界観。
  - 物語＝キャンパスライフ / 研究記録として解釈しやすく、長期的な積み上げに向く。

---

## 3. Neo Frontier Hub（ネオ・フロンティア・ハブ）

- **キーワード**: 宇宙ステーション / フロンティア開拓 / SF都市 / シアン×ゴールド
- **メインの感情トーン**: 冒険・挑戦・拡張、前向きなワクワク感
- **主な色の方向性**:
  - 背景: 深いブルーグレー〜スペースブルー
  - サーフェス（カード）: ブルーグレー〜グラファイト
  - アクセント: シアン / ティール + ゴールド（航路やUIラインをイメージ）
- **世界観イメージ**:
  - 宇宙ステーションのハブで、次のミッション（クエスト）を受注しながら新しいフロンティアへ出ていく。
  - ダッシュボードは「ミッションコントロール」、グリモワールは「航海日誌」として機能する。
- **UI・演出の方向性**:
  - カードやパネルをモジュール的に配置し、SF UI っぽいパネル感を少し意識する。
  - ボタンやタグにシアン / ティールを使い、「航路」「リンクライン」のような印象を与える。
  - ゴールド系のハイライトは、レア報酬・レベルアップ・重要アラートなど、ポジティブなイベントに限定。
- **体験としての位置付け**:
  - 新しい技術やスキルに「挑戦」していくユーザーのモチベーションと相性が良い。
  - クエストを「ミッション」として捉え、進捗がフロンティア開拓に直結していく感覚を強める。

---

## 4. 世界観テーマ運用メモ

- **共通ポリシー**:
  - 3つとも「ダーク基調＋差し色」という共通骨格を保ちつつ、雰囲気・感情トーンで差別化する。
  - テキストの可読性（コントラスト）を最優先し、世界観のために読みづらくしない。
- **実装とのひも付け**:
  - `WORLDVIEWS` の `accentColor` などと、`index.css` の `--color-accent` などのトークンを対応させる。
  - レイアウト・カード・ボタン・バッジは、可能な限りテーマトークン経由で色を決める。
- **将来の拡張余地**:
  - 各世界観ごとに、アイテム・エフェクト・バッジ（例: Arcane系アイテム、Campus系アイテム）を紐づける。
  - ナラティブジェネレータのプロンプト側でも、選択された世界観に応じた語彙・比喩を使い分ける。
