# Skill Quest AI 今後の計画

本ドキュメントは、ユーザー拡大を見据えた今後の計画を一覧化したものです。**OPS（インフラ）**、**機能・プロダクト**、**マーケティング** を統合し、タイムラインごとに実行すべき施策をまとめています。

**作成日**: 2026-02-14  
**元ドキュメント**: ユーザー拡大スケーリング計画.md、機能追加・マーケティング計画.md を統合

---

## 1. 計画の前提

### 1.1 ゴール

- ユーザー数を段階的に拡大（〜1,000 MAU → 〜10,000 MAU → 〜100,000 MAU）
- プロダクト価値の向上によるリテンション・口コミの増加
- インフラの安定とコストの可視化

### 1.2 価値提案（Value Proposition）

> タスク管理とRPG要素を融合し、AIが一人ひとりに合わせた物語を紡ぐことで、単なるTo-Do管理を超えた「冒険体験」を提供する。

## 3. コア機能の詳細（参照）

### キャラクター（ユーザー）生成の改善

現状は Genesis で AI がキャラクター（クラス、プロローグ、見た目等）を一括生成しているが、AI 依存による「微妙さ」（当たり外れ、没入感の欠如、自分事感の不足）が課題。**採用方針: テンプレート・プリセット方式**とする。

| 方針 | 説明 |
|------|------|
| **テンプレート・プリセット** | クラスやアーキタイプをあらかじめ用意し、ユーザーが選択する。AI は補助（プロローグ等の物語部分のみ）に留める |
| **実装のポイント** | ジャンルに応じたプリセット一覧、選択後のプロローグ生成（オプション）、テーマカラー等の簡易カスタム |

### AIパートナーの具現化

- テキストベース → 馴染みやすいキャラ・イラストに変更
- ジャンルに応じたデザイン。低コストでボイス追加を検討

### サンプルクエスト自動生成（目標→タスクリスト）

- Genesis で目標設定時に、AI が目標を分解して初期クエストを Quest Board に自動配置
- 例：「英語力を上げる」→「単語20個覚える」「30分リスニング」
- 初回オンボーディングの要。ゼロ状態からの心理的ハードルを下げる

### コレクションアイテム

- グリモワール生成時にレアリティ付きアイテムを獲得
- コモン / レア / スーパーレア / ウルトラレア / レジェンド

## 4. 成功指標（KPI）

| 領域 | 指標 |
|------|------|
| 獲得 | 新規サインアップ数、ランディング転換率 |
| 継続 | D1 / D7 / D30 retention |
| エンゲージメント | WAU、タスク完了数/ユーザー |
| コレクション | アイテム獲得率、レジェンド獲得ユーザー数 |
| サンプルクエスト | 自動生成タスクの完了率 |
| キャラ生成 | Genesis 完了率、やり直し率、満足度（アンケート等） |
