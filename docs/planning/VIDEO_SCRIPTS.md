# Skill Quest AI 紹介動画 台本集

60秒メイン・15秒ショート・30秒ショートの台本と、画面・演出指示をまとめたドキュメントです。  
撮影・編集時に「どこを映すか」「どう演出するか」をそのまま参照できます。

---

## 共通：使用する画面一覧

| 略称 | 画面 | 説明 |
|------|------|------|
| ログイン | ログイン/サインアップ画面 | `LoginSignupForm`。メール入力・サインアップボタン。 |
| Genesis INTRO | Genesis オープニング | 「冒険を始める」が出るイントロ画面。 |
| Genesis QUESTIONS | Genesis 質問 | 名前・目標（やりたいこと）入力フォーム。 |
| Genesis LOADING | Genesis ローディング | キャラ生成中のローディング表示。 |
| Genesis RESULT | Genesis 結果 | クラス・プロローグ・キャラ説明が表示される画面。 |
| Genesis SUGGEST | クエスト提案 | 目標から提案されたクエスト一覧（モーダル or SuggestStep）。 |
| ダッシュボード | ホーム/クエストボード | ナビ＋クエスト一覧（デイリー/習慣/To-Do）。 |
| クエスト報告モーダル | 完了報告 | タスク完了タップ後。コメント入力 → 送信 → **結果画面（物語＋XP・ゴールド・アイテム）**。 |
| パートナー | パートナーページ or ウィジェット | AIパートナーとのチャット or ウィジェット開いた状態。 |
| グリモワール | グリモワール | 過去の物語エントリ一覧・章タイトル・報酬の履歴。 |
| CTA画面 | エンドカード | アプリ名・ロゴ・「今すぐ始める」テキスト。 |

# 1. 60秒 メイン紹介動画

**総尺**: 60秒  
**目的**: 初見に「何のアプリか・なぜ他と違うか」を伝え、ストア or 登録へ誘導。

---

## 1-1. 0〜10秒：フック（課題の共感）

| 項目 | 内容 |
|------|------|
| **秒数** | 0:00〜0:10 |
| **台本（ナレーション）** | 「タスク管理アプリ、何個か試したけど……続かなかった人、多いんじゃないですか。」 |
| **画面** | **ログイン画面** を短く表示したあと、**黒 or ブランド色の背景**に切り替え。もしくは **ダッシュボードのクエスト一覧** をぼかしで一瞬見せて「いつものTODO」感を出してから切る。 |
| **演出** | ・BGM は控えめ or なし。 ・テロップで「続かなかった人、多いのでは？」を表示してもよい。 ・最後の1〜2秒で「このアプリは、ちょっと違います。」とテロップ or ナレーションを入れて次へつなげる。 |

---

## 1-2. 10〜25秒：Genesis（自分だけの物語が始まる）

| 項目 | 内容 |
|------|------|
| **秒数** | 0:10〜0:25 |
| **台本（ナレーション）** | 「目標を入れると、タスクがクエストに変わります。まずは名前と、やりたいことを入力してください。」（0:10〜0:18） 「AIが、あなただけのクラスとプロローグを生成します。」（0:18〜0:25） |
| **画面** | ・**0:10〜0:14** … **Genesis INTRO**（「冒険を始める」ボタンがある画面）→ タップする様子。 ・**0:14〜0:20** … **Genesis QUESTIONS**。名前欄・目標欄に入力していく様子（実際に文字を打つ or あらかじめ入力済みの状態を表示）。 ・**0:20〜0:23** … **Genesis LOADING**（生成中のアニメーション）。 ・**0:23〜0:25** … **Genesis RESULT**。クラス名・プロローグの冒頭テキストが読めるように映す。 |
| **演出** | ・「自分だけの物語が始まる」のテロップを RESULT の前後で表示。 ・LOADING は 2〜3 秒で十分。長くしすぎない。 ・RESULT ではプロローグの一行目が視認できる程度の長さで映す。 |

---

## 1-3. 25〜45秒：クエストクリア（一番の見せ場）

| 項目 | 内容 |
|------|------|
| **秒数** | 0:25〜0:45 |
| **台本（ナレーション）** | 「毎日のタスクは、クエストとして並びます。」（0:25〜0:28） 「ひとつ、クリアしてみましょう。」（0:28〜0:31） （タップして完了報告を送る） 「クリアするたびに、AIが紡ぐ物語と、XP・ゴールド・アイテムがもらえます。」（0:35〜0:45） |
| **画面** | ・**0:25〜0:30** … **ダッシュボード（クエストボード）**。デイリー or 習慣タブのクエスト一覧が映っている状態。一覧の「違い」が分かるように 2〜3 秒以上映す。 ・**0:30〜0:33** … いずれか **1つのクエストをタップ** → **クエスト報告モーダル**が開く。コメント欄に短く入力（または未入力のまま）→ **送信ボタンをタップ**。 ・**0:33〜0:45** … **クエスト報告モーダルの結果画面**。 ・**物語テキスト（ナラティブ）** が表示される瞬間を必ず映す。 ・**XP・ゴールド・アイテム** が表示される部分をはっきり見せる（ここを最も長く：約 10〜12 秒）。 |
| **演出** | ・「物語＋報酬が届く瞬間」で、軽く SE（クリア音・獲得音）を入れてもよい。 ・XP・ゴールド・アイテムの数字やアイコンに、短いハイライト（枠・光）を入れると「もらえた」感が伝わる。 ・ナレーションは報酬が出たあとにかぶせる形で「クリアするたびに、物語とXP・ゴールド・アイテムがもらえます」と締める。 |

## 1-4. 45〜55秒：パートナーとグリモワール

| 項目 | 内容 |

| **演出** | ・パートナーは「一緒に進める」イメージが伝わればよいので、長くても 5 秒。 ・グリモワールは「続けると残る」が分かるように、複数エントリが見えるアングルで。 |

# 4. 撮影・編集時の共通メモ

- **実機収録**: スマートフォン実機で操作しながら画面収録する想定。クエスト報告の「結果画面」は必ず実機で取得し、物語・XP・ゴールド・アイテムが同時に映るタイミングを逃さない。
- **テストデータ**: Genesis やクエストは、紹介用に「資格取得」「毎日勉強」など分かりやすい目標・クエスト名を用意しておくと伝わりやすい。
- **解像度**: 縦型（9:16）は Reels / TikTok / Shorts 用。横型（16:9）は YouTube メイン・LP 用に別尺で用意するとよい。
- **テロップ**: 各台本のキーフレーズは、ナレーションに合わせてテロップで重ねると、音なし再生でも意味が伝わる。

## 5-6. 共通メモ（拡散時）

- **CTA 統一**: 「今すぐ始める」「ストアで見る」「リンクはプロフィールから」のいずれかを動画・説明文で揃える。
- **音なし再生**: テロップで「タスク→クエスト」「物語＋アイテム」が伝わるようにする（台本どおり対応済み想定）。
- **サムネ**: 60秒は「クエスト結果画面（物語＋XP・ゴールド）」の一コマか、CTA画面。ショートは冒頭3秒の印象的なフレーム。
- **投稿時刻**: ターゲットの利用時間帯に合わせて検証（例：朝7時前後、夜21時前後）。
