## Skill Quest AI ペルソナ連動ブラッシュアップ計画（2026年3月版）

本ドキュメントは、`PERSONAS.md` で定義した 3 ペルソナ（蓮／美月／大輝）を前提に、既存アプリ（`apps/frontend`・`apps/backend`）のブラッシュアップ方針と具体的な施策案を整理したものです。  
3 月計画（`202603計画.md`）のロードマップを補完し、**「物語体験・パートナー性・成長ログ」** を最大化することを目的とします。

---

## 1. ペルソナから見た現状アプリのチャンス領域

### 1.1 共通ニーズとギャップ

- **物語の「続きが気になる」体験**
  - 既存: グリモワール／クエスト／アイテムは存在するが、**各セッションの物語セグメントと次回へのフック**の表現が弱い可能性。
  - ギャップ:
    - クエスト完了後のナラティブが「ログ」に留まり、「次のセッションへの予告」になっていない。
    - ホーム（クエストボード等）に「前回のあらすじ」「次の一手」が見えづらい。

- **AIパートナーとの関係性**
  - 既存: AIパートナー／バーページ／ペット／アイテム付与はあるが、**継続利用で関係性が変化していく実感**が不足しがち。
  - ギャップ:
    - パートナーの表情・セリフが「当日のタスク状況」に十分連動していない。
    - 美月ペルソナが好む「推しと一緒に頑張る」感情線（励まし・ねぎらい・記念日）が弱い。

- **短時間セッション設計**
  - 既存: クエストボード／Genesis での自動クエスト生成はある。
  - ギャップ:
    - 大輝ペルソナが求める「15〜20分で完結する 1 セッション」の明確な単位表示が弱い。
    - 「今日の 1 セッションを開始→完了→物語＋報酬」という導線がワンタップで始めづらい。

- **成長ログとシェア**
  - 既存: グリモワール・アイテム・パートナー好感度など成長指標は存在。
  - ギャップ:
    - 美月ペルソナが Instagram で共有したくなる「1 ページの見栄え」（スクショ前提デザイン）が十分に最適化されていない。
    - 週次／月次で振り返る「ハイライトまとめ」（物語＋実績）がない。

### 1.2 ペルソナ別の優先ポイント

- **蓮（TRPG 好きエンジニア）**
  - 欲しいもの: TRPG 的な「セッション感」「進行ログ」「週末に見返す冒険記」。
  - 改善観点:
    - クエストボードに「セッション名」「今回のシナリオ目標」を表示。
    - グリモワールを TRPG セッションログ風（セッション単位の見出し＋サマリ）に。

- **美月（成長ログ×推し）**
  - 欲しいもの: 推し（AIパートナー）との関係性、合格体験記のようなストーリー、SNS シェア。
  - 改善観点:
    - AIパートナーの台詞・表情パターンを「合格に向かうストーリーライン」に寄せる。
    - グリモワールに「今月の物語ハイライト」ビューを用意し、縦長スクショしやすいレイアウトに調整。

- **大輝（多忙 PM）**
  - 欲しいもの: 仕事用 ToDo と違う「自分の冒険」UI、短時間セッション、週末の達成感。
  - 改善観点:
    - ダッシュボード／クエストボードで「仕事の ToDo と明確に違うビジュアル」を強化。
    - 「週末セッションパック」（土日用のまとめクエスト）や「今週の冒険進捗」カードを用意。

## 2. 既存アプリのブラッシュアップ領域（画面別）

### 2.1 ランディングページ（`LandingPage.tsx`）

- **改善アイデア**
  - ペルソナ A/B/C を踏まえた 3 パターンのメッセージ・ビジュアル（例: エンジニア／資格学習／多忙ビジネスパーソン）をファーストビュー or スクロール中に差し込む。
  - 「退屈な TODO を、自分だけの物語に」に加え、**「AIパートナーと一緒に」「短時間セッションで」**など、ペルソナキーワードを強調。

### 2.2 Genesis / オンボーディング（`GenesisStep.tsx` / `GenesisStepView.tsx` / `GenesisFlowContext.tsx`）

- **改善アイデア**
  - 目標入力時に、**ペルソナに近い「プレイスタイル」選択**を追加（例: 「TRPG セッション感重視」「推しと一緒に勉強」「短時間セッションで積み上げ」）。
  - 選択したプレイスタイルに応じて、クラス・プロローグ・UI テーマの一部を変化させ、初回から「自分事感」を高める。
  - 初回クエスト自動生成を、ペルソナごとのニーズに最適化（Rust／簿記／PMP などのテンプレセット）。

### 2.3 クエストボード（`QuestBoard` 関連コンポーネント／`useQuests`）

- **改善アイデア**
  - 「今日のセッション」という単位を UI 上で明示（例: 今日の 1 セッション用クエスト束をカード表示し、完了で 1 セッション完了バッジ）。
  - クエストカードに「所要時間目安」（15 分／30 分など）を表示し、大輝向けの時間設計を可視化。
  - 蓮向けに「シナリオ名」「今回のミッション」ラベルを追加し、TRPG 的セッション感を演出。

### 2.4 グリモワール（`Grimoire` 関連 / `useGrimoire` / `routes/grimoire.ts`）

- **改善アイデア**
  - セッション単位の見出し（例: 「Session 12: 曜日＋テーマ」）を導入し、TRPG のセッションログ風に構造化。
  - 月次ハイライトビューを新設し、「今月進んだセグメント数」「代表シーン 3 つ」「レアアイテム一覧」を 1 画面にまとめる。
  - スクリーンショット前提で、縦長レイアウトと余白・タイポグラフィを Instagram 向けに最適化。

### 2.5 パートナー／ペット／アイテム（`PartnerPage.tsx` / `PetPage.tsx` / `ItemsPage.tsx` / `PartnerWidget.tsx` 等）

- **改善アイデア**
  - パートナー表情・セリフを「直近のタスク完了状況」「連続ログイン」「クエスト難易度」に応じて変化させるロジックを強化。
  - 美月向けに「合格までの道のり」ストーリーライン（章立て）を追加し、節目達成時に特別なセリフ・演出を出す。
  - 蓮／大輝向けには、「週末セッションを頑張ったときのご褒美セリフ」や「忙しい中で 1 セッションこなした時のねぎらい」など、状況別ボイス（テキスト）のバリエーションを増やす。

## 5. 運用・検証のポイント

- **計測**
  - グリモワール閲覧率・月次ハイライト閲覧率・スクショ（シェア）トライの Proxy 指標。
  - 「今日の 1 セッション」完了率・セッションあたり平均時間。
  - パートナーページ／ウィジェットの表示回数と、タスク完了数の相関。

- **定性的フィードバック**
  - Discord コミュニティで、ペルソナに近いユーザー（TRPG 好き・資格勉強垢・多忙 PM）から「物語の続き感」「パートナーの存在感」「スクショしたくなるか」を中心にヒアリング。

- **改善サイクル**
  - 3 月末時点で、小さくロールアウトした EP-45〜EP-48 の利用ログとフィードバックを確認し、4 月計画（`202604計画.md` 想定）に反映する。
