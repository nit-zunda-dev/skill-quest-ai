# Skill Quest AI コンセプト定義

アプリのコンセプトを明確にし、一貫したプロダクト判断の軸とするためのドキュメントです。  
各セクションを埋めながら、ビジョン・価値・体験を言語化してください。

---

## 1. エレベーターピッチ（30秒で伝える）

資格やスキルを伸ばしたいが、一人だと続かない人に向けて、**目標を入力するだけでタスクがクエスト（シナリオ）として一覧化**される。クエストをクリアするたびに、AIが紡ぐ**物語の1セグメント**と**XP・ゴールド・アイテム**がもらえる。**AIパートナー**と一緒にシナリオを進めるので、一人で続けるより「自分のストーリー」として習慣化できる。

---

## 2. ビジョン（目指す世界）

**「このアプリが実現したとき、世界はどうなっているか？」**

- **ユーザーにとっての変化**: 「また三日坊主」ではなく、毎日のタスクが「クエストをクリアする」体験になり、成長が物語と報酬として残る。
- **タスク管理・習慣化の文脈での変化**: 単なるTo-Do消化ではなく、ノベルゲー／TRPGのように「自分のシナリオを進める」体験として続けたくなる。一人ではなくAIパートナーと一緒にセッションを進める関係性が生まれる。
- **他にない「あれがあるから」と言える一点**: AIが一人ひとりに合わせたプロローグ・クラス・物語を紡ぎ、グリモワールに履歴が残る「自分だけのストーリーログ（セッション履歴）」がある。

**メモ・問い**:
- 5年後、ユーザーがこのアプリをどう呼んでほしいか？（例：「自分の物語を紡ぐ場所」「クエスト／シナリオ管理アプリ」）
- 競合ではなく「このアプリでないと得られないもの」は何か？ → **AIが紡ぐ物語＋ゲーム的報酬（XP・アイテム）＋パートナー**の三位一体。

---

## 3. ターゲットユーザー

### 3.1 プライマリ（最優先で届けたい人）

| 項目 | 記入 |
|------|------|
| 誰か（ペルソナの輪郭） | 20〜30代中心。資格・プログラミング・語学など「スキルを伸ばしたい」が、一人だと続かない人。 |
| どんな課題・不満を持っているか | 単純なTo-Doでは続かない。「また三日坊主」と自己嫌悪しがち。続ける動機・報酬感が足りない。 |
| どんな欲求・願望があるか | ゲーム感覚で「今すぐ」レベルアップの快感を味わいたい。習慣化を「ツール」ではなく「体験」として楽しみたい。 |
| なぜ既存のツールでは満たされないか | 一般的なタスクアプリはチェックリスト中心で、物語・成長の可視化・パートナー性がない。 |

### 3.2 セカンダリ（届けられれば嬉しい人）

- ノベルゲーやTRPGが好きで、タスク管理にその要素（物語・シナリオ・セッション感）を取り入れたい人。
- 日記・ログのように「自分の成長の記録」を残したい人。

### 3.3 想定しないユーザー（あえて狙わない）

- 純粋にタスクの整理だけが必要で、物語・報酬・パートナーに興味がない人。
- チーム・プロジェクト管理が主目的の人（個人の習慣・スキル成長に特化する）。

## 4. 価値提案（Value Proposition）

**「このアプリを使うと、ユーザーは何を得るか？」**

### 4.1 一言で

```
退屈なTODOを、自分だけの物語に。
```
（別案: 「自分専用のノベルゲー／TRPGで、習慣を続ける。」）

### 4.2 詳細（2〜3文）

- **機能的価値（何ができるか）**: 目標入力でクエスト（タスク／シナリオ）が自動提案され、完了ごとに物語セグメント・XP・ゴールド・アイテムが得られる。グリモワールにストーリーログ（セッション履歴）が蓄積する。
- **感情的価値（どう感じるか）**: 「クエストをクリアした」達成感と、AIパートナーと一緒にシナリオを進める安心感。三日坊主ではなく「主人公」としての自己像。
- **差別化（他と違う点）**: タスク＝クエスト／シナリオ、完了＝物語＋報酬、成長＝キャラクターのステータス（TRPG的な成長）。AIが一人ひとりに合わせた世界観・プロローグを紡ぐノベルゲー／TRPG的体験。

---

## 5. コア体験（ユーザーが味わう「体験」）

**「使っている最中・使った後に、ユーザーが何を体験するか」**

| 段階 | 体験の言語化 | 具体例（機能・画面レベルでなく体験で） |
|------|--------------|----------------------------------------|
| 始めるとき（オンボーディング） | 自分だけの物語が始まる感覚 | 名前・目標を入力→Genesisでクラスとプロローグが生成され、「自分のシナリオが始まる」と感じる。目標からクエストが提案され、ゼロからでも迷わない。 |
| 日々使うとき（習慣利用） | クエストボードを見るのが日課になる | タスクがクエスト（シナリオ）として並び、AIパートナーがいる。一人のTo-Doではなく「今日のセッション」として向き合える。 |
| タスクを完了したとき | クリアの瞬間に物語と報酬がもらえる | ナラティブチェックインで物語の1セグメント＋XP・ゴールド。将来的にアイテムドロップで「何が出るか」のワクワク。 |
| 成長を実感するとき | キャラとしての成長が数字と履歴で見える | レベル・XP・グリモワールの蓄積。過去のナラティブを読み返すと「自分だけのストーリーログ」として実感できる。 |
| 長く続けたとき | 続けた分だけ物語とコレクションが増える | グリモワールが厚くなり、レアアイテムが揃う。「習慣化アプリ」ではなく「自分の物語」として語りたくなる。 |

**問い**:
- このアプリの「ハイライト体験」（一番の売りにしたい瞬間）はいつか？ → **クエストをクリアした直後：物語セグメント＋報酬（XP・ゴールド・アイテム）がもらえる瞬間**。
- その体験を一言で表すと？（例：「クエストをクリアした瞬間の物語と報酬」）→ **「クエストクリアの瞬間、AIが紡ぐ物語と報酬が届く」**。

## 6. コンセプトの柱（3本柱）

コンセプトを支える「絶対に譲れない3つ」を定義する。

| # | 柱の名前 | 内容（このアプリは〇〇である） | 判断軸（これに反する機能は入れない） |
|---|----------|--------------------------------|--------------------------------------|
| 1 | 物語 | タスクは「物語の一コマ／シナリオの1シーン」であり、AIが物語として紡ぐ。単なるチェックリストではない。 | タスクが「やることリスト」のまま終わり、物語・ナラティブが伴わない機能は入れない。 |
| 2 | 成長の可視化 | XP・レベル・ゴールド・アイテムなど、TRPG的な成長が「見える」。グリモワールでストーリーログも残る。 | 成長が数値・履歴として見えない、報酬感のない設計は避ける。 |
| 3 | 一人ひとりに合わせる | 世界観・クラス・プロローグがユーザーごとに異なる。AIパートナーと一緒にシナリオを進める。 | 全員に同じテンプレートだけを押し付ける体験にはしない。（プリセット選択＋AI肉付けは可） |

**補足（柱の具体例）**:
1. **物語** … タスクは「物語の一コマ／シナリオの1シーン」であり、AIが物語として紡ぐ。単なるチェックリストではない。
2. **成長の可視化** … XP・レベル・アイテムなど、ゲーム的な成長が「見える」。
3. **一人ひとりに合わせる** … 世界観・クラス・プロローグがユーザーごとに異なる。
